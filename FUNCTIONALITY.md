# IntMap Frontend Functionality

Этот документ описывает пользовательский функционал frontend и основные сценарии работы системы со стороны клиента.

## Основные пользовательские сценарии

Frontend предоставляет следующие сценарии:

- открытие интерактивной 3D-карты здания;
- выбор здания из каталога;
- загрузка полной модели или набора этажных asset'ов;
- просмотр этажей и помещений;
- скрытие и раскрытие этажей;
- поиск маркеров;
- фокусировка камеры на точке интереса;
- просмотр деталей маркера;
- построение маршрута;
- авторизация и деавторизация;
- QR-сканирование;
- открытие карты по deep-link;
- переключение темы интерфейса.

## Сценарий старта

При открытии frontend:

1. приложение поднимает DI-контейнер;
2. получает каталог зданий с backend;
3. выбирает стартовое здание;
4. загружает 3D-модель;
5. загружает маркеры и связи;
6. показывает building title, control panel и поисковый UI;
7. запускает intro-анимацию камеры и построения здания.

Во время загрузки используется loading screen.

## Работа со зданиями

Пользователь может:

- открыть список зданий;
- выбрать другое здание;
- дождаться перестройки сцены;
- получить новую модель, новые asset'ы, новые маркеры и новый граф маршрутов.

При смене здания:

- показывается экран загрузки;
- сбрасывается часть route/selection state;
- модель и граф загружаются заново;
- затем запускается camera/building animation flow.

## Работа с этажами

Пользователь может:

- переключаться между этажами;
- скрывать неактуальные этажи;
- раскрывать и собирать этажи;
- работать с отдельными этажными asset'ами.

Важный аспект:

- при смене этажа и floor-expansion фронт синхронизирует видимость маркеров, route graph и selected route state.

## Маркеры

Поддерживаются типы:

- `MARKER`
- `FLAG`
- `GATEWAY`
- `WAYPOINT`

Что умеет frontend:

- отображать маркеры на сцене;
- скрывать их по этажу;
- скрывать или блокировать restricted объекты;
- показывать hover/selection состояния;
- открывать детали маркера;
- использовать маркер как начальную или конечную точку маршрута.

## Поиск

`SearchBar` позволяет:

- искать маркеры по имени;
- видеть список совпадений;
- выбирать результат;
- фокусировать камеру на найденной точке;
- открывать детали выбранного маркера.

Поисковая логика разделена на:

- построение индекса;
- фильтрацию;
- UI-отображение результата.

## Детали маркера

`MarkerDetailsPanel` показывает:

- название;
- тип;
- описание;
- принадлежность к этажу/помещению;
- возможные действия для фокусировки и маршрута.

Панель синхронизирована с:

- кликом по маркеру;
- результатами поиска;
- route selection flow.

## Маршрутизация

Пользователь может:

- выбрать стартовую точку;
- выбрать конечную точку;
- запросить маршрут через backend;
- увидеть путь на сцене.

Особенности:

- учитываются роли пользователя;
- gateway могут быть доступными или заблокированными;
- backend может вернуть полный или partial route;
- frontend визуализирует marker chain и path highlight.

## Авторизация

Frontend поддерживает:

- логин пользователя;
- хранение JWT в `localStorage`;
- логаут;
- обновление UI-состояния по роли;
- повторную загрузку здания при смене auth-state.

После логина/логаута:

- показывается loading screen;
- переинициализируется часть scene state;
- пересчитывается доступность помещений, asset'ов, gateway и маршрутов.

## Ограничения доступа

Frontend учитывает роли:

- `guest`
- `user`
- `admin`

Это влияет на:

- видимость ограниченных помещений;
- доступность этажей;
- доступность gateway;
- доступность asset'ов;
- допустимость построения маршрута.

## QR и deep-link

Поддерживаются два способа входа в контекст объекта:

### QR scanner

Пользователь может:

- открыть QR scanner;
- предоставить доступ к камере;
- отсканировать код;
- перейти к нужному building/flag контексту.

### Deep-link

Frontend может быть открыт по ссылке с параметрами:

- `b` — building code
- `f` — flag token / public token

После этого:

- определяется здание;
- загружается сцена;
- резолвится флаг;
- камера и UI переходят к нужной точке.

## Темы и адаптивность

Frontend поддерживает:

- светлую тему;
- тёмную тему;
- desktop layout;
- mobile layout;
- мобильный QR flow;
- адаптацию модальных окон под portrait и landscape.

## Что особенно важно для демонстрации

Самые показательные сценарии:

1. открытие здания;
2. переключение этажей;
3. поиск маркера;
4. открытие деталей;
5. построение маршрута;
6. скрытие restricted зон у гостя;
7. логин и повторный показ с расширенными правами;
8. QR/deep-link сценарий.

Этот набор покрывает большую часть пользовательского функционала frontend.
